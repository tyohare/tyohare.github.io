HTTP Redirect :

  • We can control a parameter that will redirect to a resource of our choice

HTTP Parameter Injection:

  • Injecting a parameter into a request that wasn't intended to be there

HTTP Parameter Pollution:

  • When we exploit the unknown behavior of having a parameter set 2 times in a request (Will it use the first or last occurrence of the parameters value?)

Cross Site Websocket Hijacking

  • Similar to CSRF where we can listen in on established websocket connections that have no random tokens present.

Cache Control:

max-age - Time in seconds where we can use a cached item (freshness)
no-cache - Response can still be cached, but must be revalidated with the server before each reuse
no-store - Response will not be cached by browser or intermediate caches (Proxy,CDN)
