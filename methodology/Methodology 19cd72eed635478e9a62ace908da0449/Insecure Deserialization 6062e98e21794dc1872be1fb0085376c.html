Insecure Deserialization

How to identify:

  • Verbose errors
  • files on the application
  • Verbose headers
  • Obvious serialized data (JSON, XML)
  • Decoding
  • Tool output

PHP

  • Serialized inhuman readable string format

  • Look for serialize() and unserialize()

Java:

  • Often binary serialized
  • Usually start with the same bytes (AC ED for hexadecimal, rO0 in Base64)
  • Look for Java.io.serialize, ReadObject(), and InputStream

Methods of exploitation:

  • Directly modifying human readable strings
  • serializing our own object for binary serialized objects (ysoserial, PHPGCC)
  • Create a gadget chain from scratch
