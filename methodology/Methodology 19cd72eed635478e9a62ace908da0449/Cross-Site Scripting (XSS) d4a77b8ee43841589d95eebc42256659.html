Cross-Site Scripting (XSS)

  • https://cheatsheetseries.owasp.org/cheatsheets/XSS_Filter_Evasion_Cheat_Sheet.html
  • XSS impact: take actions in the context of effected users..
  • Reflected → server reads and responds. It reflects back on the page.
    • Alert script example, Stealing Cookies
    • Usually requires client side interaction (Social engineering, Clicking a link)
    • The JS is reflected back into the website HTML, JS, CSS, etc
  • Stored → inject a payload and we can leave and come back it will still be there
    • Comment section, blog posts
    • This stores the payload in a database and every time the database is loaded into the webpage, executes the JS
    • Does not require client side interaction
  • DOM → exploiting DOM client side
  • Reflected and DOM require social engineering
  • Inspecting anything with input or parameters
  • XSS Payloads and try them through BURP intruder.
  • Preventing XSS:
    • Input Validation
      • Comparing input against whitelist
      • Only accepted characters or input types are accepted as input
    • Output Encoding
      • < becomes &lt;
      • <script> becomes &lt;script>
      • HTML entity encoding for HTML and UTF-8 / unicode for Javascript
      • Context aware output encoding ensures output is returned as literal, not executable
    • Filtering
      • <script> become script
    • Sanitization
      • Combination of escaping, filtering, validation

Methodology:

  • Start with adding random input into fields and submitting.
  • Open inspect HTML and see where the input is reflected…
    • This is important to know what kind of XSS we need to perform (HTML,JS,CSS)
  • Use this knowledge to craft XSS payload
