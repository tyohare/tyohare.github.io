Attacking Authorization

Two main vulnerabilities:

API1: Broken Object Level Authorization (BOLA)

GET /api/v2/10101-1010-10101010/location
  • We can fuzz the ‘10101-1010-10101010’ id or look around the site to see if there are any ids being used for other features

API5: Broken Function Level Authorization (BFLA):

  • Performing actions you're not allowed to perform.
  • We can find through source code review, documentation
  • Example: Being able to grab all users from an endpoint. 
You have:
GET /api/v2/users/1
you find vulnerability at:
GET /api/v2/users

OR

You have:
POST /api/v2/user/delete/1
you find vulnerability at:
GET /api/v2/admin/delete/1
