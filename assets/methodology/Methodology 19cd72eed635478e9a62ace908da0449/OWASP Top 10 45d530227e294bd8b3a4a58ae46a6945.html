2017 List:

  1. Injection Attacks:
    • SQL Injections
    • malicious SQL commands injected into a database
  • Classic SQL Login Injection
admin' OR 1=1 --

  • Repeater shows a lot more information than manual testing

  • Blind SQL injection: Testing commands and observing how the website reacts.
    test' (sleep 5)
test' (sleep 10)
  • SQL Injection Defense:
    • Parameterized Statements
      • Ensure inputs (parameters) are used safely in SQL statements
      • Example: “SELECT * FROM users WHERE email =?”;
      • Example: “SELECT * FROM users WHERE email =’” + email + “’”;
    • Sanitizing Input

  1. Broken Authentication
    • credential stuffing
    • brute force attacks
    • sending hundred of requests
    • default or well known passwords
    • Forgot password processes
    • MFA
    • Session IDs in URL, Rotating Session IDs (session fixation)
    • Testing:
      • invalid email or password statements (information disclosures)
      • Session fixation
      • Forgot password process
      • default credentials
      • MFA
    1. Sensitive Data Exposure
      • Personally identifiable information exposed
      • clear text passwords
      • information in source code
      • Don’t store sensitive data unnecessarily
      • Strong passwords
      • Testing
        • Enumeration: discovering items that should not be exposed
        • response headers and cookies in burp
        • securityheaders.com
        nmap --script=ssl-enum-ciphers 445 tesla.com
        • attackable ciphers
    1. XXE (XML External Entities)
      • Attacking systems that parse XML input
      • Dos, LFI, RCE etc.
      • XML basics
      <?xml version="1.0" encoding="ISOBLAH"
<!DOCTYPE gift [
	<!ENTITY from "C:/Heath&Amber">
]>

<gift>
	<To>Frnak</To>
	<From>&from;</From>
	<Item>Cards</Item>
</gift>
      • google XXE Payloads
        • Payloads all the things
      • We can put malicious characters in our entities that are banned in normal use
      • File upload features
      • Bypassing whitelisted extensions
    1. Broken Access Control
      • User gets access to somewhere they shouldn't
      • trying to get admin areas
      • IDORs → seeing someone else's information
      • inspect element on hidden hold values
      • Forging feedback
    1. Security Misconfigurations
      • default credentials on webpages
      • Unnecessary features with vulnerabilities, things exposed
      • bad settings
      • Use best practices! changes default passwords!

    1. XSS (Cross site scripting)
      • Reflected → server reads and responds. It reflects back on the page.
        • Alert script example, Stealing Cookies
      • Stored → inject a payload and we can leave and come back it will still be there
        • Comment section, blog posts
      • DOM → exploiting DOM client side
      • Reflected and DOM require social engineering
      • Inspecting anything with input or parameters
      • XSS Payloads and try them through BURP intruder.
      • Preventing XSS:
        • Encoding
          • < becomes &lt;
          • <script> becomes &lt;script>
        • Filtering
          • <script> become script
        • Validating
          • Comparing input against whitelist
        • Sanitization
          • Combination of escaping, filtering, validation
    1. Insecure Deserialization
      • Taking data into a serialized format… XML Json Binary etc
      • Deserialization is opposite of that process
      • We can send malicious serialized commands and it will be deserialized and attacked
      • Don’t accepted Serialized objects from untrusted sources
    1. Using component with known vulnerabilities
      • Deprecated versions with CVEs
      • Always be patching
    1. Insufficient logging and monitoring
      • Logging and monitoring failed logins, suspicious behavior.
      • Are you getting caught while pentest scans are happening?
