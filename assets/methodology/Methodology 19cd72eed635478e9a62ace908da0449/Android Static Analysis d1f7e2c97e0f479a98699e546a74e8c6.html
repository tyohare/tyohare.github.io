Android Static Analysis

  • Android is based on linux
    • Therefore, it uses linux filesystem

Andorid is compiled by android runtime

  • dalvik was the original android runtime
  • Every android app is sandboxed and not allowed to talk to other apps
  • Every app has a user for the application

/data/app/com.example.app - generic app data

/data/data/com.example.app - runtime data storage

/mnt/sdcard/Android/data/com.example.app - external stored location for runtime

/data/data/com.example2.app - different app requiring a different user

Android profiles .. Have some isolated aspected: Kids, mode, guest user primary user, secondary users, work profiels

Decompiling an APK

  • We can use APKTool
apktool d application-release.apk

Pulling an APK:

ex: injuredandroid

in android studio: 

adb shell (gets shell for phone)
pm list packages | grep “injured”
pm path <package name>
exit
adb pull <path to package> package-pulled.apk

Android Manifest.xml:

  • every app has it
  • Can be found in JADX Gui
  • Resources → AndroidManifest.xml
  • applications foundations are defined (Check these)
    • minSDKVersion
    • permissions
    • hardcoded strings
    • Activities
      • Exported activities
    • Content Providers

permissions:

  • What the app needs access to…
    • camera, bluetooth, internet, contacts, etc

Activities:

  • UI elements or different screens in the app
  • intent filters allow us to protect unauthorized access to activities
  • an Exported=”True” intent can be viewed outside the app

Content providers

  • used to serve data from your app to another app
