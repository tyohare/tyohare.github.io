Single Sign On (OAuth/SAML/OIDC)

  • Created to share information between applications

OAuth grant types:

Authentication Bypass via OAuth Implicit Flow:

- When we capture the SAML flow we may find authentication parameters provided to the server
 after authenticating with the IDP.
- We can edit these authentication parameters to the user we want to login as
	and forwarding it to the server.

OAuth account hijacking via redirect_uri:

- If we send a malicious URL or IFrame payload to an admin user,
- We edit the redirect URI variable to point to our malicious server where the admin users token can be leaked
- From there we can take that token to and use it to hijack the admin account

SAML:

Principle (user) → SP (service provider)

SP (Sends AuthnRequest ) - > user

user (AuthnRequest) → Idp (identity provider)

User authenticates

IDP (SAML Response) → user

user (SAML Response) → SP

User is redirected to page.

We can manipulate the SAML response to the SP.

If we manipulate the AuthnRequest we are attacking the IDP.

Attack types include:

  • editing signature, assertion times, user data in SAML response.
  • Removing the signature from SAML response
  • Changing the assertion content
  • SAML response signed with self-signed cert
  • SAML response should only be accepted one time (Replaying)
    • Is it in the duration of NotBefore and NotOnOrAfter?
  • SAML response passing…
